JWT Structure

        
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9  ← Header (Base64url encoded)
.
eyJzdWIiOiJhbGljZSIsInJvbGUiOiJ1c2VyIn0 ← Payload (Base64url encoded)
.
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c ← Signature (HMAC-SHA256)

Header  = { "alg": "HS256", "typ": "JWT" }
Payload = { "sub": "alice", "role": "user", "exp": 1700000000 }
Signature = HMAC_SHA256(base64(header) + "." + base64(payload), SECRET_KEY)
        

      

      

        
How Requests Work

        
GET /api/protected HTTP/1.1
Host: localhost:5000
Authorization: Bearer <your_jwt_token>

Server receives → extracts token → verifies signature → reads claims → responds
        

      

    


    
    

      

        

          
✅ Best Practices

          
• Use HTTPS always
• Short expiry (15 min)
• Refresh token rotation
• Strong secret key
• Validate all claims
• Use HS256 minimum
• Hash passwords (bcrypt)
• Revoke on logout
          

        

        

          
❌ Common Mistakes

          
• Storing in localStorage
• Long expiry times
• Weak secret keys
• No token revocation
• Trusting payload blindly
• Using "none" algorithm
• Sending over HTTP
• No signature check
